CADENCE — CASE STUDY

Cadence is a hybrid glove + camera MIDI controller that turns real hand motion into expressive music control. It fuses glove sensors (ESP32) with webcam hand tracking (MediaPipe) to drive notes, CCs, and a live harmonic bed.

What it does

  • Glove sensors (ESP32): 4 flex sensors, thumb touch/pressure ADC, hall sensors, IMU.
  • Webcam tracking (MediaPipe): finger bend + hand position from a single hand.
  • Gesture velocity triggers note-on/off (fast vs slow EMA) for “pluck” style playing.
  • Thumb pressure → MIDI CC27, wrist pitch → CC28 + octave bias.
  • Hand X/Y → CC25/26; hall sensors can act like sustain (CC64).
  • Built-in harmonic engine: Markov/chord-library pad on MIDI ch2 while melody plays on ch1.

How we built it

  • ESP32 streams compact CSV sensor data over serial/Bluetooth (supports multiple formats).
  • Python engine: robust serial reader + MediaPipe + Madgwick AHRS for stable pitch.
  • Sensor fusion + smoothing (EMAs/deadbands) for responsiveness without jitter.
  • MIDI out via mido + rtmidi (“GestureHand MIDI”) → works with any DAW/synth.

Challenges

  • Noisy sensors + dropped frames → needed strong filtering + fallback behavior.
  • Latency vs stability → tuned fast/slow EMA and attack detection.
  • Sensor direction mismatches → per-sensor inversion + calibration.
  • IMU drift/alignment → calibration, deadbanding, smoothing to avoid twitchy octave jumps.
  • Musical feel → deadbands for CC spam, auto note-off aging, careful thresholds.

What we’re proud of

  • Natural gesture-to-note triggering using bend velocity (not static thresholds).
  • True hybrid tracking: glove continues when vision fails; camera adds nuance when available.
  • Full performance surface: CC27/28 + CC25/26 + sustain makes it feel like an instrument.
  • Plug-and-play MIDI: virtual port works anywhere with zero special integration.

What we learned

  • Filtering + deadbands can make cheap sensors feel “premium.”
  • Calibration is UX — a wizard turns a demo into an instrument.
  • Fusion beats perfection: two imperfect systems together are more reliable.
  • Real-time systems need guardrails: auto-release, reconnection, fallback modes.

What’s next

  • Save/load user calibration + mapping profiles.
  • More gestures: pinch/tap/poses (mute, latch chord, arpeggiate).
  • Gestures influence harmony (tension, inversion, voicing).
  • Wireless polish + packaging: UI for status, mapping, and synth presets.

AM SIGNAL PROCESSING SYSTEM — ES 152

In ES 152: Circuits, Devices, and Transduction, I designed and simulated a complete multi-carrier AM transmitter and receiver system in LTspice. The project implemented amplitude modulation using analog multipliers, frequency-selective filtering, and system-level impedance-aware design.

System Architecture

The system encodes two independent audio signals onto separate carrier frequencies (500 kHz and 1500 kHz) using amplitude modulation. The modulated signals are summed onto a single transmission channel and later separated and recovered through high-order filters.

  • Audio signals: 20 kHz and 10 kHz
  • Carrier frequencies: 500 kHz and 1500 kHz
  • Transmission channel: summed multi-carrier signal
  • Receiver: frequency-selective filtering + demodulation

Analog Multiplier Implementation

To implement amplitude modulation, I built an analog multiplier using log/exp op-amp stages leveraging the exponential I-V relationship of diodes. The architecture used:

  • Log amplifier stages
  • Inverting summing amplifier
  • Exponential (anti-log) stage
  • Impedance buffering between stages

Because the multiplier only operates correctly for positive inputs, DC offsets were introduced and later removed using high-pass filtering. This required careful frequency-domain reasoning to eliminate unwanted low-frequency terms introduced by biasing.

3rd-Order Filter Design

Separating closely spaced carrier frequencies required higher-order filters. I designed cascaded 3rd-order low-pass and high-pass filters using a hybrid RLC + RC architecture.

  • Target cutoff frequencies: 600 kHz (LPF) and 1400 kHz (HPF)
  • -60 dB/decade roll-off achieved via cascading
  • RLC resonance used to compensate RC attenuation
  • Unity gain at cutoff via resonance tuning (+3 dB compensation)

AC sweeps (1 kHz – 1 GHz) were used to verify magnitude and phase response. The design required balancing cutoff sharpness, resonance gain, and impedance interactions between stages.

System Integration & Impedance Management

The full system integrated transmitter, summing amplifier, antenna driver, and receiver filters. Each stage was designed with input/output impedance in mind to prevent loading effects and frequency shifts.

  • Buffered cascaded stages where required
  • Designed antenna driver for 50 Ω load
  • Verified time-domain modulation and recovery
  • Ensured scalability for additional carriers

Key Takeaways

  • High-order filters require both magnitude and impedance awareness.
  • Resonance can intentionally shape system-level gain.
  • Biasing tricks often introduce frequency-domain artifacts that must be filtered.
  • Modular analog blocks allow scalable communication architectures.
